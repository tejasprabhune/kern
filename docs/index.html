kern

Typst math, in the browser. A drop-in alternative to KaTeX.

Why kern

Typst's math syntax is meaningfully cleaner than LaTeX's. Writing frac(a, b) instead of \frac{a}{b}, or sum_(i=0)^n instead of \sum_{i=0}^{n}, adds up over a long document. But until now there was no way to render Typst math on the web without shipping a 10MB WASM binary.

kern is a pure TypeScript renderer that parses Typst's math syntax and emits MathML or HTML+CSS. The core bundle is under 20KB gzipped. No WASM, no web workers, no fonts to bundle.

How it works

kern runs a hand-written recursive descent parser over Typst math source, producing a typed AST. The AST is then walked by a renderer that emits either native MathML (supported in all major browsers since 2023) or a KaTeX-compatible HTML+CSS fallback using KaTeX's web fonts.

The pipeline is: source text → lexer → parser → AST → MathML string. Sub-millisecond for typical expressions.

Install

npm install kern
# or
pnpm add kern

Or via CDN (jsDelivr):

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/kern/styles/kern.css">
<script src="https://cdn.jsdelivr.net/npm/kern/dist/kern.min.js"></script>

Quick start:

import { render } from 'kern';

render('sum_(i=0)^n i = frac(n(n+1), 2)', document.getElementById('math'));

Status

kern is pre-1.0. All v1 features from Typst's math mode are implemented: fractions, roots, attachments, matrices, style functions (cal, bb, frak), the full symbol table, auto-render, and both MathML and HTML/CSS output.

See the roadmap for what comes next (visual regression suite, CI, docs site improvements).
